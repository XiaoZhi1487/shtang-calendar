# 沙塘圩日历 · 版本更新与打包流程

> 版本基准：v1.0.5
> 操作系统：Windows
> 构建环境：Node.js + Android Studio JBR (Java 21)

---

## 一、版本号更新（每次发布必须改）

需要同时更新 **两处** 版本号，确保应用内和系统应用信息里显示一致：

### 1. 应用内版本号（用户在「我的 → 检查更新」看到的版本）

文件：`src\store\userStore.ts`

```typescript
const APP_VERSION = '1.0.5';   // ← 改成新版本号，例如 1.0.6
```

### 2. Android 系统版本号（手机下载/安装时和应用信息里看到的）

文件：`android\app\build.gradle`

```gradle
defaultConfig {
    applicationId "com.shtang.calendar"
    minSdkVersion rootProject.ext.minSdkVersion
    targetSdkVersion rootProject.ext.targetSdkVersion
    versionCode 5            // ← 每次 +1（整数，用于系统判断版本新旧）
    versionName "1.0.5"      // ← 与上面保持一致，例如 "1.0.6"
    ...
}
```

> **关键点**：
> - `versionCode`：**必须是整数，且每次递增**，Android 系统用它判断「更新」，相同就提示「已安装相同版本」
> - `versionName`：展示给用户看的字符串，必须和 `APP_VERSION` 保持一致

## 二、手动构建步骤（标准流程）

### 步骤 1：构建前端 Web 资源

在项目根目录（`archive2`）执行：

```powershell
npm run build
```

成功后会看到：
```
vite v6.4.3 building for production...
✓ 1666 modules transformed.
✓ built in 10.16s
```

产物在 `dist\` 目录。

### 步骤 2：同步前端资源到 Android 工程

```powershell
npx cap sync
```

成功后会看到：
```
√ Copying web assets from dist to android\app\src\main\assets\public
√ Creating capacitor.config.json
√ Sync finished in 0.181s
```

### 步骤 3：配置 Java 环境（Gradle 打包需要 Java 11+）

系统默认的 Java 可能是 1.8，不够构建要求。临时切换到 Android Studio 自带的 JBR：

```powershell
$env:JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"
$env:PATH = "$env:JAVA_HOME\bin;" + $env:PATH
```

验证 Java 版本：
```powershell
java -version
```
应该看到 `openjdk version "21.0.5"` 或类似。

### 步骤 4：用 Gradle 打包 APK

进入 `android` 目录执行：

```powershell
cd android
.\gradlew.bat assembleDebug
```

首次构建会下载依赖，大约 2-5 分钟。成功结尾：
```
BUILD SUCCESSFUL in 27s
87 actionable tasks: 47 executed, 40 up-to-date
```

APK 产物路径：
```
android\app\build\outputs\apk\debug\app-debug.apk
```

### 步骤 5：复制并重命名到项目根目录（可选，方便上传）

```powershell
Copy-Item "android\app\build\outputs\apk\debug\app-debug.apk" ".\沙塘圩日历_v1.0.5.apk"
```

把 `1.0.5` 换成当前版本号。

## 三、一键自动化脚本（推荐，以后直接用这个）

项目根目录已准备了 PowerShell 脚本 `一键打包.ps1`，**右键 → 使用 PowerShell 运行** 或在 PowerShell 终端执行：

```powershell
.\一键打包.ps1
```

脚本会自动完成：
1. 显示当前配置的版本号（供你核对）
2. `npm run build`
3. `npx cap sync`
4. 配置 JAVA_HOME 并执行 `gradlew assembleDebug`
5. 把 APK 复制到项目根目录并重命名为 `沙塘圩日历_vX.X.X.apk`

> 如果脚本执行失败，终端会停留在错误位置，方便查看原因。

## 四、更新发布到数据库（可选，配合「检查更新」功能）

如果你希望用户在应用内点「检查更新」时能提示新版本，需要在后端数据库的 `app_version` 表插入一条记录。

后端服务（server.js）启动后，也可以通过以下方式添加（示例命令，实际可直接用数据库工具插入）：

| 字段 | 值示例 | 说明 |
|------|--------|------|
| version | `1.0.5` | 新版本号 |
| release_note | `1. 修复登录后数据不同步的问题\n2. 优化版本号显示` | 更新说明 |
| download_url | `https://xxx/沙塘圩日历_v1.0.5.apk` | APK 下载地址（蓝奏云等） |
| created_at | `CURRENT_TIMESTAMP` | 自动生成 |

## 五、常见问题

### Q1：Gradle 报错 "Could not resolve com.android.tools.build:gradle" / "Dependency requires at least JVM runtime version 11"

**原因**：系统默认 Java 是 1.8，而当前 Gradle 插件要求 Java 11+。

**解决**：按步骤三切换 Java 环境，或把系统环境变量 `JAVA_HOME` 永久设置为 `C:\Program Files\Android\Android Studio\jbr`。

### Q2：`gradlew.bat` 找不到命令

**原因**：没有在 `android` 子目录执行，或 PowerShell 不识别当前目录脚本。

**解决**：用 `.\gradlew.bat`（前面加 `.\`）而不是直接写 `gradlew.bat`。

### Q3：PowerShell 里写 `a && b` 报错

**原因**：PowerShell 不支持 `&&` 作为命令链，用 `;` 分隔或者分别执行两条命令。

### Q4：手机安装时提示「已安装相同版本」

**原因**：忘记更新 `versionCode`，或者 `versionCode` 没递增。

**解决**：每次打包前确保 `versionCode` **严格 +1**（例如从 5 改成 6）。

### Q5：构建成功后，APK 在哪里？

固定路径：
```
android\app\build\outputs\apk\debug\app-debug.apk
```
