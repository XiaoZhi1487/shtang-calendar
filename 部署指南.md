## 🌐 部署后端到公网（免费方案）

### 推荐方案：使用 Render 免费部署

**Render** 是一个免费的云平台，可以部署 Node.js 应用！

#### 步骤 1：准备你的代码

确保你的项目已经提交到 GitHub/GitLab。

#### 步骤 2：注册 Render 账号

1. 访问：https://render.com
2. 使用 GitHub 账号注册（免费）

#### 步骤 3：创建新 Web Service

1. 点击 "New +" → "Web Service"
2. 选择你的 GitHub 仓库
3. 配置如下：
   - **Name**: `shtang-calendar-server`（随便起）
   - **Region**: `Singapore`（新加坡，离中国近）
   - **Branch**: `main`（你的主分支）
   - **Runtime**: `Node`
   - **Build Command**: `npm install`
   - **Start Command**: `node server.js`

#### 步骤 4：配置环境变量

在 Render 的 "Environment" 页面添加：

```
DB_HOST=mysql6.sqlpub.com
DB_PORT=3311
DB_USER=xiaott
DB_PASSWORD=b1y6ukxRrVGopQH7
DB_NAME=shatang_userdata
PORT=10000
```

#### 步骤 5：部署！

点击 "Create Web Service"，等待 2-3 分钟部署完成！

#### 步骤 6：获取你的公网地址

部署成功后，Render 会给你一个地址，类似：
`https://shtang-calendar-server.onrender.com`

## 🔧 更新前端配置

### 步骤 1：修改 API 地址

打开文件：`src/config/api.ts`

把 `PROD_API_BASE` 改成你的 Render 地址：

```typescript
// 生产环境 - 改成你的 Render 服务器地址
const PROD_API_BASE = 'https://shtang-calendar-server.onrender.com/api';
```

### 步骤 2：重新构建 APK

```bash
npm run build
# 然后用 Capacitor 打包 APK
```

## 💡 其他免费部署方案

### 方案 B：Vercel（免费，无服务器限制）
- 官网：https://vercel.com
- 优点：国内访问速度快
- 缺点：需要把后端改成 Serverless Functions

### 方案 C：Railway（免费 5 美元/月额度）
- 官网：https://railway.app
- 优点：部署简单，支持 MySQL
- 缺点：免费额度有限

### 方案 D：自己的服务器（如果你有）
- 阿里云、腾讯云、华为云等
- 最便宜的云服务器大概 50-100 元/月
